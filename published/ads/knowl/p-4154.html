  1. The result of doing arithmetic modulo \(n\) is always an integer between 0 and \(n-1\text{,}\) by the Division Property. This observation implies that \(\{0, 1,\dots, n-1\}\) is closed under modulo \(n\) arithmetic.
  2. It is always true that \(a +_n b \equiv (a + b) \pmod{n}\) and \(a\times_n b \equiv (a \cdot b) \pmod{n}\text{.}\) For example, \(4 +_7 5 = 2 \equiv 9 \pmod{7}\) and \(4 \times_7 5 = 6 \equiv 20 \pmod{7}\text{.}\)
  3. We will use the notation \(\mathbb{Z}_n\) to denote the set \(\{0, 1, 2,. . ., n-1\}\text{.}\) One interpretation of this set is that each element is a representative of its equivalence class with respect to congruence modula \(n\text{.}\) For example, if \(n=7\text{,}\) the number \(1\) in \(\mathbb{Z}_7\) really represents all numbers in \([1]={1+7k : k\in \mathbb{Z}}\text{.}\) In doing modular arithmetic, we can temporarily replace elements of \(\mathbb{Z}_n\) with other elements in their equivalence class modulo \(n\text{.}\)
